# 协议，不是屏幕（protocol, not screen）

> 编排者要控制 worker agent，最顺手的路是它的界面：抓屏读状态、send-keys 发指令。
> 这条路我们走到头了——2026-07 把整条 TUI 控制车道裁撤，换成一条 duplex lane、
> 三引擎各跑官方原生协议。本文记录这次升级的数据、理念与架构判断。
> 一句话：**控制面要长在协议上，不是屏幕上。**

## 抓屏为什么必死

- **状态靠猜**：从 pane 文本正则推断完成，WAITING 被误读成 DONE（实证单日 4 次假 DONE）；
  glyph 与布局随 CLI 版本漂移，一轮常规升级同时弄坏三个引擎的抓屏链路。
- **注入靠赌**：send-keys 实测仅 ~70-80% 送达——弹窗吃掉 Enter、bracketed-paste 丢尾、
  长文本损坏，一次卡死 24 分钟。
- **根因是结构性的**：屏幕是给人的 UI，不是接口。机器把它当控制面，等于寄生在一条
  无契约的表面上——上游每次 UI 改版都是你的 breaking change，而你连版本号都拿不到。

修补这条路的努力（更聪明的正则、更稳的注入时序）全是在给寄生关系续命。出路只有换宿主：
三大引擎如今都有官方 headless 双工面（JSON-RPC / stream-json），直接消费引擎自己的
结构化事件流 + 进程退出码 + 交付物文件，状态从"猜"变成"判"。

## 换成什么

一条 lane、五个动词：`start | steer | status | watch | stop`。每个引擎跑自己的原生协议长驻，
tmux 降级为进程保活；能力差异不分叉车道，由接口诚实拒绝或降级并指路。真相只来自
typed exit code、协议帧、rc 文件与交付物新鲜度——**永不抓屏**。

## 数据（同一批生产席位，新旧两代实测）

| 派发确认 | fire-and-forget，送达不可证 | goal 帧 correlated ack，送达机械可证 |
| 轮中纠偏 | 不可能（运行中拒收；实证 workaround 是往 worker 文件树里塞信号文件） | `steer --now` 同轮吸收；idle 续轮免重派仪式 |

## 理念一：steer 是一等公民

**编排回路的半径由纠偏成本决定。** 纠偏 = 一整轮时，goal 合同必须一次写全、评审只能压到
轮末、错误发现即返工——整个方法论被迫围绕"别出错"设计。纠偏 ≈ 0 时，合同可以更薄，
发现问题当场注入，多轮 fix→review→fix 链在同一会话上下文里滚动。**协议给的不只是可靠性，
是编排自由度。**

行业在同一时间走向同一结论：主流 harness 都把 steering 做成了一等 API——区分轮中送达与
轮末送达、区分可中断与不可中断的工具调用，甚至给 steer 带乐观并发控制（expectedTurnId 的
CAS 语义）。CAS 是给机器 supervisor 防竞态用的，人类按 Esc 就够了——**steering 的目标调用者
已经从人变成程序**。coding agent 的本体正在变成 headless、长驻、协议驱动的进程，CLI 降级为
前端之一。

## 理念二：typed 真相

沉默不是完成，idle 不是交付，turn 边界不是任务终态。控制面输出一个有限的 typed 词汇
（DONE / FAILED / WAITING-INPUT / IDLE-NO-DELIVERABLE / ENGINE-SILENT / BUDGET-EXHAUSTED…），
每个词绑定处置动作；判不出时诚实报 ENGINE-SILENT，不猜。且 watcher 裁决是线索不是判决：
DONE 收货前仍核正向交付证据——typed 词汇约束的是控制面不说谎，不豁免编排者验收。

## 理念三：控制面输出有界

编排者的上下文是整个编排系统最稀缺的资源。任何回传无界的控制面，迟早把编排者炸成
只会滚日志的终端（88k tokens 事故）。摘要有界、raw 全量落文件，是硬设计不是优化。

## 架构判断：四层栈与我们押的位置

行业正收敛到四层：**模型 API → harness 内核 → 运行时控制协议 → 编排器**。工具接入有 MCP，
跨组织协作有 A2A，而 agent 的**运行时控制面**——steer / status / 完成语义这一层——还没有
事实标准，三家引擎三套方言。agentctl 押的正是这一层的编排器侧：一张引擎无关的统一动词表 +
typed 状态词汇 + 交付物门，方言差异收进适配层、由接口诚实拒绝。我们的目标是把这套动词表
慢慢长成智能体编排的一种规范——谁的引擎有原生协议，就接谁。

## 诚实边界

omp / codex 腿有两日生产重载实测；claude 腿过了 live e2e 闭环、真实生产单仍待补；
BLOCKED 阻塞协议的 live fire 待验。规范的可信度来自边界诚实，不来自宣称全绿。

---

一句话：**屏幕是人的界面，协议是机器的界面——控制面长错了地方，纪律和聪明都救不回来；
而编排的自由度，是 steer 的成本给的。**
