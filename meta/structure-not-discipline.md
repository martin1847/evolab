# 结构而非自律（structure, not discipline）

> 同一条纪律有三级载体，可靠性递增、也该沿此升级：
> **L0 散文提醒 → L1 强制层（hook 拦截）→ L2 设计消灭（错误无处发生）。**
> 来源：多编排者实战一周内四例同向演化（2026-07 沉淀）。

## 三级阶梯

- **L0 散文**：写在 skill/文档里的"记得做 X / 别做 Y"。要"读过+记得+对的时刻想起"，三缺一即失效；
  不 fire 的散文是**净负债**（稀释真信号、制造已治理假象）——要么升级、要么删。
- **L1 强制层**：tool-call 层 hook 在动手那一刻 DENY / 提醒。比散文硬，但仍是**事后拦截**：
  规则与实现两处维护会漂移、matcher 有覆盖缺口、且每次拦截都消耗一轮往返。
- **L2 设计消灭**：改结构让错误**没有发生的表面**。没有可漂移的规则、没有要维护的拦截、没有要记住的步骤。

## 实证（每条都走过完整阶梯或跳级）

| 病 | L0 | L1 | L2（终态） |
|---|---|---|---|
| 派发后漏挂 watch | "记得 watch" 散文 | guard 注入提醒 | dispatch `--goal` 融合自动 watch——**单独步骤不存在了** |
| 信件状态腐烂 | "记得更新 status" | — | 砍 status 字段，**状态即位置**（inbox/archive） |
| wiring 手抄漂移 | 文档示例 + "别抄错" | 发布门校验 matcher | settings 成 **hooks.json 真源的生成物**，没人手编辑 |
| 解释器前缀押注 PATH | "别加 python 前缀" | E2E 断言拒前缀 | 脚本 shebang + 执行位，**裸路径是唯一形态** |
| 双 flag 依赖脚枪 | "`--watch` 必与 `--goal` 同用" | guard 豁免逻辑 | 删掉 `--watch`——"必须与另一 flag 同用的 flag"本身就是待删的设计 |

## 判据

- **每条 guard 都是一张设计欠条**：它拦得住错误，但说明结构仍允许错误发生。guard 积累 = 设计债信号。
- **何时值得从 L1 升 L2**：slip 高频、或 guard 自身开始长条件分支（豁免/特例）、或错误源于"固有多步"——
  多步能融合成一步的，融合；可变字段能变成位置/派生的，砍掉；抄本能变成真源生成物的，生成。
- **何时停在 L1**：错误面在人/模型的判断里而非结构里（如"证据够不够"），结构消灭不了判断——那就让 hook
  把判断材料推到决策点。
- 反向裁剪：升到 L2 后，**回头删掉为它服务的 L0 散文和 L1 特例**（保留一行"为什么这里没有 X"的墓志铭即可），
  否则三层同时活着又成新的漂移面。

## L1 出口契约：电必带路（deny-with-directions）

拦截只做了一半——被电的一方还需要当场知道**哪条规则、为什么、去哪读、怎么走正路**。每条 DENY 文案
必带三件：一句 why（带实证日期更佳）+ 正路命令 + **owning doc 指针**（`Read: <skill>/<path>.md §…`）。
先例：rustc error code + `--explain`、ESLint/clippy 每条违规带 rule 文档 URL；poka-yoke 只盖"硬闸"半边、不含回链。

由此得**软硬分工闭环**：文档写方向与判据、保持简洁（写得再细也不会被 100% 执行，细节只稀释真信号）；
承重规则全部下沉 hook；教学发生在电击那一刻。**双向指针、两边都瘦**——doc 里被强制的规则只留一行
"已由 X hook 强制"，hook 文案不复述论证、指回 doc。文档给建议，hook 给底线，deny 给路。

自指：本契约自己就是机器门——`test/hook-deny-pointer.test.sh` 扫描全部 shipped hook 源码，断言每条
DENY 含 `.md` 指针（覆盖 f-string / 隐式拼接形态，扫描器负例已验）；新加 DENY 忘带路，门自动红。

## 关系

- 是 `agency-by-reversibility` 的工程面兄弟：那条管"谁可以做"，这条管"错误能不能发生"。
- 是「不 fire 的散文是净负债」razor 的正向出口：razor 说删或硬化，本条说硬化之后还有下一站。
