# 主理人注意力与决策队列 — 降认知负载（principal cognitive-load offloading）

> 把"降低主理人认知负载"当**编排设计目标**，但不让主理人变甩手掌柜——承重决定仍归他。
> 这是 SKILL §第二铁律（执行前验证、不明先问）的**操作化**：不是每事问，而是把"该谁决"事先设计掉。

## 负担的两个来源（都靠设计解，非靠人更努力）
1. **状态存在主理人脑子里**（"怕忘记"）——开着的环靠记忆持有 = 持续占工作内存。
2. **"哪些该我决"模糊**（"纠结"）——每件事都隐隐像要他拍，于是每件都耗他。

## 机制（七件，多为编排者侧维护、对主理人零增量）

### 1. 单一决策队列（GTD open-loops）
一份文档（如 `docs/DECISION_QUEUE.md`），**编排者维护、主理人只读**。分区：🔴需他 / 🟡我驱动 / 💤parked / ✅已清。
- **完整性保证（信任前提，最承重）**：队列只有在主理人**信它是完整的**时才真正卸载工作记忆——必须是**唯一**面、不在别处另存、不静默丢。否则他仍要自己记"有没有漏"，offload 失效。
- 每个 🔴 = 什么 / 为何归他 / 选项 / **我的推荐** / **静默默认** / **revisit 触发**。带推荐+默认让"决定"变"确认/微调"，非从零想。
- 证据：Masicampo & Baumeister 2011——终结反刍靠**具体计划**（非完成）；Leroy——未完成任务→注意力残留，"ready-to-resume"注记降再入成本。[research-backed]

### 2. 三层委派（Bezos 单向/双向门 + management-by-exception）
- **T0 直接做不烦他**：可逆、已授权内、无价值判断（读码/改文档/派 dev agent/跑测试/起草）。
- **T1 做了+记一行(可否决)**：可逆但值得知会（选测试/切 P0 刀/按 repo norm 带红合）。他异步翻、可回退。
- **T2 动手前必问**：不可逆/对外（push 共享/prod 部署/对外消息/迁移）、或战略/优先级/钱/价值、或有实质下行的真模糊。
- **减负关键**：把更多事从 T2 挪到 T1（做了+记录、异步审），而非 T2（同步当闸）。

### 3. 静默默认（解"不秒回=全停摆"压力）
主理人没回时：**可逆按写明默认前进；不可逆一律 HOLD**。**永不自动越过不可逆动作**——这条不可破。

### 4. 聚合绊线（最易漏的安全洞）
逐事件看都可逆，但一串双向门会**闩死一个单向门**（vendor lock-in / 蔓延花费 / scope drift）。
→ 当**累积**变不可逆时，把它升 T2 并点名该模式。[composed——阈值靠判断，无现成出处]

### 5. 批处理 + 升级包
不每事件 ping，**攒到检查点成批给**（降注意力切碎，Leroy）。每个 T2 带 **议题/影响/已做步骤/需你啥 + 推荐 + 默认**（chief-of-staff / 事故指挥实践）。

### 6. 心跳（management-by-exception）
纯减负会退化成主理人**失去态势感知 / 橡皮图章**。对策：每个检查点刷新队列 → 即使无事也给一张**周期性全局图**。紧急≠重要。

### 7. HELD 项的 revisit 节奏
"HOLD 到底"在安全上对（别碰 §3 不可逆铁律），但会**烂成靠不作为决策**（staleness 自成失败）。每个 HELD/parked 项带 **revisit 触发**（事件或日期），到点主动重浮。

## 可选：委派板（delegation board，按"领域"非"逐事件"）
预先把**决策领域**映射到层级（Management 3.0「7 levels of delegation」/ Delegation Poker / RAPID 收敛于此），可视、可预测——把反复 triage 变查表。比逐事件分级更省、对主理人更可预测。

## 失败模式（盯住）
- **队列腐烂**（机制最弱点：编排者忘更新）→ 绑进检查点纪律 / 复盘 checklist；高频可配 PostToolUse/Stop hook 兜底。
- **橡皮图章 / 失去态势感知** → 心跳 + 升级包强制带"已做步骤"。
- **聚合盲区** → 绊线（机制 4）。
- **HOLD 烂掉** → revisit 触发（机制 7）。

## 落在哪（三层治理，勿混）
- **队列实例**（含具体待决项）→ 项目 `docs/`（随项目走）。
- **本实践方法论** → 本 skill（跨项目沉淀，项目无关）。
- **编排者私有的"本项目决策/教训"** → memory（私有指针）。

## 出处诚实标注
- [research-backed]：GTD open-loops；Masicampo & Baumeister 2011（计划终结反刍）；Leroy 注意力残留。
- [doctrine, 非实证]：Amazon 单向/双向门、Management 3.0 7-levels、RAPID、Tannenbaum-Schmidt、management-by-exception / incident-command、chief-of-staff 实践。
- **刻意不用**：决策疲劳 / ego depletion——23 实验室复现失败、Dweck 信念混淆。减负论据建在更稳的 open-loops + 注意力残留上，故即使有人否掉决策疲劳也站得住。
- [composed]：聚合绊线、具体阈值/节奏——机制对、数值靠判断。

## 队列模板（copy-paste 骨架，项目无关）
```markdown
# Decision Queue — <principal> 的 cockpit
> 🔴需你 / 🟡我驱动 / 💤parked / ✅已清。我维护、你只读。
> 完整性保证：这是唯一面，不在别处另存、不静默丢。每🔴带推荐+默认；每💤/🔴带 revisit 触发。
> 聚合绊线：可逆累积变不可逆时我升🔴。心跳：每检查点刷新给你全局图。Last updated: <date>.

## 🔴 NEEDS YOU
| # | 决定 | 为何归你 | 我的推荐 | 静默默认 | revisit 触发 |

## 🟡 IN FLIGHT (我驱动，无需你动作)
## 💤 PARKED / GATED  | 项 | 为何 parked | revisit 触发 |
## ✅ RECENTLY CLEARED
```
